# Architecture Decisions (Summary)

This document summarizes the Architecture Decision Records (ADRs) captured in `Architecture/ADR`.

- ADR 0001 — Record architecture decisions: use ADRs for documenting decisions (accepted).
- ADR 0002 — Use multiple projects: multi-project solution structure (accepted).
- ADR 0003 — Use Clean Architecture boundaries: enforce Domain → Application → Infrastructure → API (accepted).
- ADR 0004 — Adopt Aspire for orchestration: use AppHost for service orchestration (accepted).
- ADR 0005 — Integration style: REST for commands/queries and Server-Sent Events (SSE) for real-time updates (accepted).
- ADR 0006 — Update authentication provider: Keycloak vs Okta PoC (proposed, pending).
